## 執行結果與說明

**常見的 EC2 連線問題診斷步驟**:

1. **檢查實例狀態** (需要 AWS 控制台):
   - 實例是否正在運行 (running)
   - 狀態檢查是否通過
   - 系統可達性檢查

2. **檢查網路設定** (需要 AWS 控制台):
   - 安全群組是否允許 SSH (埠 22)
   - 網路 ACL 設定
   - 路由表設定

3. **基本網路測試** (可在本地執行):
   ```bash
   # 需要實例的公共 IP
   ping -c 4 <PUBLIC_IP>
   telnet <PUBLIC_IP> 22
   ```

**請提供實例的公共 IP 地址以繼續測試**
